# Rattle 앱 기능 명세

> 식당/매장 관리 앱 — Expo SDK 55 + React Native + TypeScript + Supabase

---

## 1. 인증 (Authentication)

### 1.1 로그인
- 이메일 + 비밀번호 로그인 (Supabase Auth)
- 앱 재시작 시 세션 자동 복원

### 1.2 회원가입
- 이메일 + 비밀번호로 신규 계정 생성

### 1.3 매장 선택 (`/(auth)/select-store`)
- 로그인 후 보유 매장 목록 카드 표시
- 현재 선택된 매장 체크마크 표시
- 기존 매장 탭 → 즉시 전환 후 홈 이동
- "새 매장 추가" 버튼 → 매장명 입력 → 생성 후 자동 전환

### 1.4 로그아웃
- 세션 종료, 저장된 마지막 매장 ID 초기화

### 1.5 회원 탈퇴
- `delete_user_data` RPC 호출로 매장·데이터 전체 삭제
- 탈퇴 후 로그인 화면으로 이동

---

## 2. 다중 매장 관리

- 한 계정에 여러 매장 생성·보유 가능
- `AsyncStorage`에 마지막 선택 매장 ID 저장, 앱 재시작 시 복원
- 프로필 화면 → "매장 전환" 메뉴로 언제든 전환
- `switchStore(storeId)` — 전환 시 모든 데이터 해당 매장 기준으로 갱신

## 3. 대시보드 (홈 탭)

### 3.1 요약 카드 (4개)
| 카드 | 내용 | 이동 |
|------|------|------|
| 이번 달 발주 | 당월 발주 건수 | 발주 탭 |
| 품절 임박 | `current_stock ≤ min_stock` 항목 수 | 재고 탭 |
| 레시피 | 등록된 레시피 수 | 레시피 탭 |
| 평균 마진율 | 전체 레시피 평균 마진율(%) | 레시피 탭(마진 정렬) |

### 3.2 예상 재고 소진량 (오늘)
- 오늘 영업일 내 완료된 Toss Place POS 주문 집계
- 판매된 메뉴명과 동명 레시피의 재료 수량 × 판매 수량 계산
- 소진량 상위 5개 재료 표시 (재료명 + 수량 + 단위)
- POS 주문 데이터 없으면 섹션 미표시

### 3.3 빠른 실행
- 납품서 스캔 → 발주 탭 이동
- 발주 추가 → 발주 등록 화면
- 재고 확인 → 재고 탭
- POS 연동 → POS 탭

### 3.4 최근 활동
- 최근 발주 5건 + 최근 식자재 등록 5건 통합, `created_at` 기준 최신 5건 표시
- 활동 없으면 빈 상태 안내

## 4. 발주 관리 (Orders)

### 4.1 발주 목록 (`/(tabs)/orders`)
- 무한 스크롤 페이지네이션 (PAGE_SIZE=20)
- 발주일 내림차순 정렬
- 상태 배지: 대기중 / 확인됨 / 납품완료

### 4.2 납품서 OCR 스캔
- 카메라로 납품서 촬영 → Supabase Edge Function(OCR) 파싱
- 파싱 결과 검토 화면 (`/orders/ocr-review`)에서 항목 확인 후 발주 등록

### 4.3 발주 등록 (`/orders/new`)
- 거래처명, 발주일, 품목(식자재 + 수량 + 단가) 입력
- `create_order_with_items` RPC로 발주 + 항목 원자적 생성

### 4.4 발주 상세 (`/orders/[id]`)
- 발주 정보 및 품목 목록 조회
- "납품 완료" 처리: `deliver_order` RPC → 재고 자동 증가 + `last_price` 업데이트

---

## 5. 재고 관리 (Stock)

### 5.1 재고 목록 (`/(tabs)/stock`)
- 카테고리별 그룹핑 표시
- 무한 스크롤 페이지네이션
- 품절 임박 항목 경고 표시 (`current_stock ≤ min_stock`)

### 5.2 재고 등록/편집 (`/stock/new`, `/stock/[id]`)
- 식자재명, 카테고리, 현재 재고, 단위, 최소 재고, 최근 단가, 용량 입력

### 5.3 카테고리 관리
- 카테고리 이름 변경, 삭제(미분류로 이동)

### 5.4 미분류 일괄 카테고리 지정 (BulkCategoryModal)
- 미분류 항목 수 배너 표시 (실제 DB 조회 기반)
- 모달에서 전체 미분류 항목 체크리스트 + 카테고리 입력
- 선택 항목 일괄 카테고리 업데이트

---

## 6. 레시피 관리 (Recipes)

### 6.1 레시피 목록 (`/(tabs)/recipes`)
- 카테고리별 그룹핑 표시
- 정렬: 최신순 / 마진율순 (URL 파라미터 `sort=margin`)
- 무한 스크롤 페이지네이션

### 6.2 레시피 등록 (`/recipes/new`)
- 메뉴명, 카테고리, 판매가, 재료(식자재 + 사용량) 입력
- `create_recipe_with_ingredients` RPC로 원자적 생성
- 원가·마진율 자동 계산: `last_price / container_size × quantity` 합산

### 6.3 레시피 상세/편집 (`/recipes/[id]`)
- 레시피 정보 조회 및 수정
- 재료 추가/제거

### 6.4 카테고리 관리
- 카테고리 이름 변경, 삭제(미분류로 이동)

### 6.5 미분류 일괄 카테고리 지정
- 재고와 동일한 BulkCategoryModal 방식

### 6.6 POS 카탈로그 가져오기 (CatalogImportModal)
- Toss Place 카탈로그 품목 체크리스트 (카테고리별 그룹핑)
- 선택 확인 → `upsertRecipesFromCatalog`:
  - 동명 레시피 존재 시: `selling_price`만 업데이트 (재료 보존)
  - 미존재 시: 신규 insert (`cost=0`, `margin_rate=0`)

## 7. Toss Place POS 연동 (`/(tabs)/pos`)

### 7.1 가맹점 연동 설정 (`/settings/pos-sync`)
- Toss Place 가맹점 신청 폼 (사업자번호, 대표자명, 연락처, 주소 등)
- 신청 정보 저장 후 연동 대기

### 7.2 매출 조회
- 영업일 기준 일별 요약 목록 (날짜, 총 매출, 주문 수)
- 영업일 기준: `stores.closing_time`(기본 23:00) 기준 전날 ~ 당일

### 7.3 수동 동기화
- 특정 날짜 동기화 버튼 → `syncByDate(date)`
- `upsert_toss_order_with_items` RPC로 주문 + 상세항목 원자적 Upsert

### 7.4 자동 동기화
- 앱 포그라운드 복귀 시 `autoSync()` 자동 실행
- 오늘 데이터 없을 때만 동기화 (중복 방지)

### 7.5 카탈로그 동기화
- "카탈로그 동기화" 버튼 → `toss_catalog` 테이블 저장
- 동기화 완료 후 CatalogImportModal 자동 오픈

### 7.6 날짜별 상세 (`/pos/[date]`)
- 해당 날짜 상품별 판매 수량 및 매출 표시
- 카테고리별 그룹핑

## 8. 프로필 설정 (`/settings/profile`)

### 8.1 매장 정보 수정
- 매장 이름 변경
- 마감 시간 변경 (HH:MM 형식, POS 영업일 기준으로 사용)

### 8.2 매장 전환
- `/(auth)/select-store` 화면으로 이동

### 8.3 로그아웃
- 확인 다이얼로그 → 세션 종료 → 로그인 화면 이동

### 8.4 회원 탈퇴
- 경고 다이얼로그 → `delete_user_data` RPC → 전체 데이터 삭제 → 로그인 화면 이동

## 9. 공통 사항

### 데이터 흐름
```
컴포넌트 → custom hook → lib/services/ → Supabase
```
- 컴포넌트에서 Supabase 직접 호출 금지

### 에러/로딩 처리
- `LoadingSpinner` — 전체 화면 로딩
- `ErrorMessage` — 에러 메시지 + 재시도 버튼

### 색상 시스템
- `constants/colors.ts`의 `Colors.*` 사용, 하드코딩 금지

### RLS (Row Level Security)
- 모든 테이블에 RLS 적용, `store_id` 기준 데이터 격리

### 트랜잭션 보장 RPC
| RPC | 역할 |
|-----|------|
| `create_order_with_items` | 발주 + 항목 원자적 생성 |
| `create_recipe_with_ingredients` | 레시피 + 재료 원자적 생성 |
| `deliver_order` | 납품 완료 + 재고 증가 + last_price 갱신 |
| `upsert_toss_order_with_items` | Toss 주문 + 상세항목 원자적 Upsert |
| `delete_user_data` | 회원 탈퇴 시 전체 데이터 삭제 |
